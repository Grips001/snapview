Create `claude-integration/SKILL.md` with this exact content:

```markdown
---
name: snapview
description: Capture a screen region and inject the screenshot into Claude's context. Use when you need to see the user's screen, UI, terminal output, or visual content. Suggest this when the user says things like "let me show you", "look at this", "check this UI", or similar.
disable-model-invocation: true
allowed-tools: Bash(snapview *)
---

Launch the screen capture UI so the user can select a region of their screen.

Steps:
1. Tell the user: "Launching capture UI..."
2. Run: `snapview` using the Bash tool
3. If exit code is 0: the file path is printed to stdout. Read that file immediately using the Read tool and continue the conversation with the screenshot in context.
4. If exit code is 2 (user cancelled): respond with "Screenshot cancelled. Want to try again?"
5. If exit code is 1 (error): respond with "Screenshot failed. You may need to check permissions or try again."
6. If the command times out (30s): respond with "Screenshot capture timed out. The capture window may need to be closed manually."
```

Create `scripts/snapview-autotrigger.js` as a Node.js script (NOT bash — cross-platform per Pitfall 5 in research). The script:

1. Reads stdin to completion (all input arrives as a single JSON blob from Claude Code)
2. Parses the JSON input — extracts `last_assistant_message` field
3. Checks if `SNAPVIEW_AUTO_TRIGGER` env var is set to `"1"` — if not set or not `"1"`, exits 0 immediately (global toggle)
4. Attempts to parse `last_assistant_message` as JSON — if it contains `{"snapview_capture": true}`, proceed with capture
5. Also scans `last_assistant_message` for the pattern `{"snapview_capture":true` as a substring (Claude may embed the JSON signal within prose text)
6. If trigger detected: runs `snapview` binary via `child_process.execFileSync('snapview', [], { timeout: 32000, encoding: 'utf8' })` — timeout 32s (under the 35s hook timeout, above the 30s Electron hard-exit)
7. On success (exit 0): outputs JSON to stdout: `{"decision":"block","reason":"Screenshot captured. Read the file at: <filepath> and continue the conversation with the screenshot in context."}`
8. On exit code 2 (user cancelled): exits 0 silently (no block, let Claude continue)
9. On error/timeout: outputs JSON to stdout: `{"decision":"block","reason":"Screenshot capture failed or timed out. Continue without the screenshot."}`
10. If no trigger signal found: exits 0 silently

Add `#!/usr/bin/env node` shebang at top. Use only Node.js built-ins (`child_process`, `process`). No external dependencies.

Handle the stdin read with:
```javascript
let input = '';
process.stdin.on('data', (chunk) => { input += chunk; });
process.stdin.on('end', () => { /* main logic here */ });
```

Create `scripts/postinstall.cjs` as a CommonJS module exporting `install()` and `uninstall()` functions. Also auto-runs `install()` when executed directly (for npm postinstall).

Constants:
```javascript
const CLAUDE_DIR = path.join(os.homedir(), '.claude');
const SETTINGS_PATH = path.join(CLAUDE_DIR, 'settings.json');
const SKILLS_DIR = path.join(CLAUDE_DIR, 'skills', 'snapview');
const SKILL_PATH = path.join(SKILLS_DIR, 'SKILL.md');
const HOOKS_DIR = path.join(CLAUDE_DIR, 'hooks');
const HOOK_FILENAME = 'snapview-autotrigger.js';
const HOOK_PATH = path.join(HOOKS_DIR, HOOK_FILENAME);
```

**`install()` function:**
1. Check Claude Code is installed — check BOTH `fs.existsSync(CLAUDE_DIR)` AND whether `claude` binary is in PATH via `child_process.execFileSync('which', ['claude'])` (or `where` on Windows: `process.platform === 'win32' ? 'where' : 'which'`). If dir doesn't exist but binary does, create the dir with `fs.mkdirSync(CLAUDE_DIR, { recursive: true })`. If neither exists, print `"Claude Code not detected. Install Claude Code first, then re-run."` to stderr and `process.exit(1)`.
2. Create skills directory: `fs.mkdirSync(SKILLS_DIR, { recursive: true })`
3. Copy SKILL.md: read from `path.join(__dirname, '..', 'claude-integration', 'SKILL.md')` and write to `SKILL_PATH`
4. Create hooks directory: `fs.mkdirSync(HOOKS_DIR, { recursive: true })`
5. Copy hook script: read from `path.join(__dirname, 'snapview-autotrigger.js')` and write to `HOOK_PATH`. On non-Windows, `fs.chmodSync(HOOK_PATH, 0o755)`.
6. Read-modify-write `settings.json`:
   - Read existing file if it exists, parse JSON (if parse fails, start with `{}`)
   - Ensure `settings.hooks` exists (default `{}`)
   - Ensure `settings.hooks.Stop` exists (default `[]`)
   - Remove any existing snapview entry: filter Stop array entries where `JSON.stringify(entry).includes('snapview-autotrigger')` returns false
   - Compute hook command path: `HOOK_PATH.replace(/\\/g, '/')` (forward slashes for Windows compatibility per Pitfall 4)
   - Push new entry: `{ hooks: [{ type: 'command', command: 'node ' + hookCommandPath, timeout: 35 }] }`
   - Ensure `settings.env` exists (default `{}`)
   - Set `settings.env.SNAPVIEW_AUTO_TRIGGER = '1'` (auto-trigger enabled by default)
   - Write back: `fs.writeFileSync(SETTINGS_PATH, JSON.stringify(settings, null, 2) + '\n', 'utf8')`
7. Print `"Snapview installed successfully!"` to stdout

**`uninstall()` function:**
1. Remove skill directory: `fs.rmSync(SKILLS_DIR, { recursive: true, force: true })`
2. Remove hook file: `fs.rmSync(HOOK_PATH, { force: true })`
3. Read-modify-write `settings.json`:
   - Read and parse (if missing or unparseable, skip)
   - Filter out snapview entries from `settings.hooks.Stop` (same filter as install)
   - If `settings.hooks.Stop` is now empty, delete the `Stop` key
   - If `settings.hooks` is now empty, delete the `hooks` key
   - Remove `settings.env.SNAPVIEW_AUTO_TRIGGER` if it exists
   - If `settings.env` is now empty, delete the `env` key
   - Write back
4. Print `"Snapview uninstalled successfully!"` to stdout

**Auto-run block at bottom:**
```javascript
if (require.main === module) {
  install();
}
```

**Exports:**
```javascript
module.exports = { install, uninstall };
```

Use only Node.js built-ins: `fs`, `os`, `path`, `child_process`. No external dependencies.
