Create `src/renderer/app.ts` — the complete renderer logic for the overlay capture UI.

**Type declaration for the preload bridge** (at top of file):
```typescript
declare global {
  interface Window {
    snapviewBridge: {
      getSources(): Promise<{ id: string; thumbnail: string }[] | { permissionDenied: true }>;
      captureRegion(rect: { x: number; y: number; width: number; height: number }): Promise<{ filePath: string }>;
      cancel(): Promise<void>;
    };
  }
}
```

**State machine:**
- Track `currentPhase: 'selecting' | 'previewing'`
- Track `selectionRect: { startX, startY, endX, endY } | null`
- Track `screenImage: HTMLImageElement | null` (the full screen thumbnail loaded as an Image)

**Initialization (on DOMContentLoaded):**
1. Get DOM references: `canvas`, `ctx` (2d context), `hintBar`, `previewPanel`, `previewImage`, `btnApprove`, `btnRetake`, `permissionDialog`, `btnOpenSettings`
2. Set `body.classList.add('selecting')` for crosshair cursor
3. Call `window.snapviewBridge.getSources()`
4. If result has `permissionDenied: true`:
   - Show `#permission-dialog` (`display: flex`)
   - Hide `#hint-bar`
   - Attach click handler on `#btn-open-settings` that calls `window.snapviewBridge.cancel()`
   - Return early (do not set up canvas listeners)
5. If sources array is empty: log error to console, call `window.snapviewBridge.cancel()`, return
6. Take the first source's thumbnail (data URL string), create an `Image()`, set `src` to the data URL
7. On image load: set canvas width/height to `window.innerWidth` and `window.innerHeight`, draw the initial dimmed overlay

**Drawing functions:**

`drawDimOverlay()`:
- Draw the screen image stretched to fill canvas: `ctx.drawImage(screenImage, 0, 0, canvas.width, canvas.height)`
- Draw dim layer over entire canvas: `ctx.fillStyle = 'rgba(0, 0, 0, 0.45)'; ctx.fillRect(0, 0, canvas.width, canvas.height)`

`drawSelection(startX, startY, endX, endY)`:
- Calculate rect: `x = Math.min(startX, endX)`, `y = Math.min(startY, endY)`, `width = Math.abs(endX - startX)`, `height = Math.abs(endY - startY)`
- If width < 5 or height < 5: don't draw selection (too small, likely a click)
- Clear and redraw dim overlay via `drawDimOverlay()`
- Clear the selection area to show full brightness: Use a clipping approach — save context, begin path for selection rect, clip, redraw screen image at full brightness, restore context
- OR simpler approach: after drawing dim overlay, use `ctx.clearRect(x, y, width, height)` then redraw just the selection portion of the screen image with `ctx.drawImage(screenImage, x, y, width, height, x, y, width, height)` — this avoids the dim layer inside the selection
- Draw 2px white border: `ctx.strokeStyle = 'rgba(255, 255, 255, 0.8)'; ctx.lineWidth = 2; ctx.strokeRect(x, y, width, height)`

**Mouse handlers (on canvas):**

`mousedown`:
- If `currentPhase !== 'selecting'`: return
- Record `startX = e.clientX`, `startY = e.clientY`
- Set `isDragging = true`

`mousemove`:
- If not `isDragging`: return
- Set `endX = e.clientX`, `endY = e.clientY`
- Call `drawSelection(startX, startY, endX, endY)`

`mouseup`:
- Set `isDragging = false`
- Calculate rect width/height
- If width < 5 AND height < 5: treat as click-without-drag — call `window.snapviewBridge.cancel()` and return (per UI-SPEC: cancel on click-without-drag)
- Else: transition to previewing state

**Transition to previewing:**
- Set `currentPhase = 'previewing'`
- Remove `selecting` class from body (removes crosshair cursor)
- Hide `#hint-bar` (`display: none`)
- Calculate the selection rect: `{ x: Math.min(startX, endX), y: Math.min(startY, endY), width: Math.abs(endX - startX), height: Math.abs(endY - startY) }`
- Create a temporary canvas to crop the preview from the screen image:
  ```typescript
  const cropCanvas = document.createElement('canvas');
  cropCanvas.width = rect.width;
  cropCanvas.height = rect.height;
  const cropCtx = cropCanvas.getContext('2d')!;
  cropCtx.drawImage(screenImage, rect.x, rect.y, rect.width, rect.height, 0, 0, rect.width, rect.height);
  ```
- Set `previewImage.src = cropCanvas.toDataURL('image/png')`
- Show `#preview-panel` (`display: flex`)

**Button handlers:**

`btnApprove` click:
- Calculate the selection rect (same as above)
- Call `await window.snapviewBridge.captureRegion(rect)`
- The main process handles stdout output and app.quit()

`btnRetake` click:
- Set `currentPhase = 'selecting'`
- Add `selecting` class back to body
- Hide `#preview-panel`
- Show `#hint-bar`
- Clear canvas and redraw initial dim overlay via `drawDimOverlay()`
- Reset `selectionRect = null`

**ESC handler (on document keydown):**
- If `e.key === 'Escape'`: call `window.snapviewBridge.cancel()`
- This works in both `selecting` and `previewing` states

**CRITICAL implementation notes:**
- The canvas `drawImage` for the screen source uses the data URL thumbnail from the main process. The Image element is loaded from this data URL.
- Selection coordinates are in CSS pixels (canvas is sized to `window.innerWidth/Height`). The main process handles DPI scaling.
- Do NOT import from 'electron' — all communication via `window.snapviewBridge`
- Handle the case where getSources returns an array with proper null/empty checks
