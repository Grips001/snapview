# Phase 1 вҖ” UI Design Contract: Capture Engine

> Visual and interaction contract for the Electron overlay renderer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | RESEARCH.md вҖ” vanilla TS renderer, no UI framework |
| Preset | not applicable | Greenfield Electron app вҖ” no component library |
| Component library | none | Canvas-based overlay; no DOM component tree |
| Icon library | none | No icons in Phase 1 UI (text labels only) |
| Font | system-ui, -apple-system, sans-serif | Default OS font; no web font load penalty |

**Rationale:** This is an Electron BrowserWindow renderer with a fullscreen canvas. No React, no shadcn, no Radix. The only CSS targets are the canvas element, the hint bar, and the preview panel. shadcn gate does not apply.

---

## UI States

Phase 1 has exactly three UI states. The design contract covers all three.

| State | Trigger | What the User Sees |
|-------|---------|-------------------|
| `selecting` | App launch | Fullscreen dimmed overlay with crosshair cursor + hint text |
| `previewing` | Mouse-up after drag | Dimmed overlay remains; centered preview panel appears with approve/retake |
| `cancelled` | ESC key or Retake | Overlay closes; process exits cleanly |

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Hint | 13px | 400 (regular) | 1.4 | "Drag to select / ESC to cancel" overlay text |
| Label | 14px | 600 (semibold) | 1.0 | Button labels: "Send to Claude", "Retake" |
| Heading | 16px | 600 (semibold) | 1.2 | Preview panel title: "Capture Preview" |
| Body | 14px | 400 (regular) | 1.5 | macOS permission denial message body text |

**Notes:**
- 3 distinct sizes only: 13, 14, 16. Two weights: 400 and 600.
- 13px hint text is below the standard 14px floor because it must be unobtrusive on a fullscreen overlay. Source: CONTEXT.md "visible but unobtrusive."
- All text must be antialiased via `-webkit-font-smoothing: antialiased` in the renderer CSS.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(0, 0, 0, 0.45)` | Overlay dim layer covering the entire screen canvas |
| Secondary (30%) | `rgba(15, 15, 15, 0.92)` | Preview panel background вҖ” near-black, solid |
| Accent (10%) | `rgba(255, 255, 255, 0.9)` | Selection border, button primary background ("Send to Claude") |
| Destructive | `rgba(200, 60, 60, 0.9)` | Not used in Phase 1; reserved |

**Selection cutout:** The selected region is rendered at full brightness by clearing the dim layer with `ctx.clearRect()`. The region itself shows the unmodified screen content вҖ” no color applied.

**Selection border:** 2px solid `rgba(255, 255, 255, 0.8)` drawn around the selection rectangle. Source: RESEARCH.md Pattern 8. This is the only accent usage on the selecting state.

**Hint bar:** Background `rgba(0, 0, 0, 0.60)` вҖ” darker than the overlay to ensure legibility. Text color `rgba(255, 255, 255, 0.90)`.

**Preview panel:**
- Background: `rgba(15, 15, 15, 0.92)` with `border-radius: 8px`
- Border: `1px solid rgba(255, 255, 255, 0.12)`
- "Send to Claude" button: accent white background (`rgba(255, 255, 255, 0.90)`) with dark text (`#111`)
- "Retake" button: transparent background with white border (`rgba(255, 255, 255, 0.40)`) and white text

Accent reserved for: selection border only, primary "Send to Claude" button background, and macOS permission action link.

## Interaction Contract

### Selecting State

| Interaction | Behavior |
|-------------|----------|
| Cursor on overlay | `cursor: crosshair` applied via CSS on the canvas element |
| Mouse drag (mousedown вҶ’ mousemove) | Rubber-band rectangle drawn via canvas; dim layer clears inside selection rect on each frame |
| Mouse-up | Selection committed; app transitions to `previewing` state; crosshair cursor removed |
| ESC key | `app.quit()` called; exit code 2; no stdout output |
| Click without drag (mousedown == mouseup position) | Cancel вҖ” treat as ESC (user likely mis-clicked); exit code 2; no stdout. Source: CONTEXT.md "Claude's Discretion вҖ” click-without-drag behavior" вҖ” choosing cancel over full-screen capture for safety |

### Previewing State

| Interaction | Behavior |
|-------------|----------|
| "Send to Claude" click | `captureRegion(rect)` IPC call; PNG written to temp dir; file path emitted to stdout; exit code 0 |
| "Retake" click | Transitions back to `selecting` state; canvas redrawn with fresh dim layer |
| ESC key | `app.quit()` called; exit code 2; no stdout output |
| Preview image | Scaled to fit panel (max 480px wide, max 320px tall); `object-fit: contain`; aspect ratio preserved |
| Preview panel position | Horizontally and vertically centered in the BrowserWindow |

### macOS Permission Denial

| Interaction | Behavior |
|-------------|----------|
| Status is `denied` | Native-style dialog/modal displayed (not toast) over the overlay; shows "Screen Recording Permission Required" heading, body text, and "Open System Settings" button |
| "Open System Settings" click | `shell.openExternal()` to macOS privacy settings; app quits with exit code 1 |
| Dismiss without granting | App quits with exit code 1; error message emitted to stderr |

**Source:** CONTEXT.md "macOS permission denial UX (dialog vs toast)" is Claude's discretion вҖ” choosing modal dialog over toast because it blocks the broken capture flow. Toast would let the user attempt capture again, which will fail.

## Implementation Notes for Executor

1. **No CSS framework.** Write a single `src/renderer/styles.css`. Use CSS custom properties for the color values declared above.
2. **Canvas operations are the primary UI.** The dim + cutout effect is rendered frame-by-frame in `mousemove`. Do not attempt to implement this with DOM elements.
3. **Preview panel is a DOM element** positioned over the canvas via `position: fixed`. It is shown/hidden via CSS `display: none / flex`. It is NOT a second BrowserWindow. Source: RESEARCH.md Open Question 3 вҖ” same-window panel chosen to avoid two-window IPC complexity.
4. **macOS permission dialog** is a DOM modal (not a native dialog). Show it as `position: fixed; z-index: 9999` centered in the window, with a semi-transparent backdrop.
5. **Hint bar** is a DOM element (`position: fixed; bottom: 0`), not drawn on the canvas. This keeps the canvas free for selection rendering.
6. **ESC key handler** must be registered on `document` with `keydown` event, not on the canvas.
7. **Cursor:** `cursor: crosshair` on `body` for the selecting state; remove it when transitioning to previewing.
